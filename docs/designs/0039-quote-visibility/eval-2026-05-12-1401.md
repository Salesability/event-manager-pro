## Codex Adversarial Review

### Pass 1 — 2 Medium (self-resolved)

**Medium 1 — Archived dealers render as `—` in the composer Dealer field.**

> `/quotes/[id]` passes only `dealerId` into `initial`; the composer label looks up that id in `dealerOptions` (built from `loadDealers()` which excludes archived) and falls back to `—`. Recommended fix: include `dealerName` in `InitialQuote`.

**Medium 2 — Read-only non-draft quotes show recomputed lines, not the persisted snapshot.**

> `loadQuote` exposes persisted `subtotal/tax/total/inputs` but not `lineItems`; the edit page passes only `inputs` and `tax`; the composer renders `computeQuote(inputs, catalog, taxOverride)`. Non-draft rows are immutable, so a read-only sent/accepted/declined quote can display values that differ from the locked row if catalog pricing changed.

### Fixes applied (Codex write-capable rescue)

- `queries.ts` — `Quote` type + projection + `mapRow` extended with `lineItems: ComputedLine[]` (cast via `unknown` per repo jsonb convention).
- `queries.test.ts` — `baseRow` fixture got `lineItems: []`; `loadQuote` test now asserts a non-empty `lineItems` round-trip.
- `quote-composer.tsx` — `InitialQuote` extended with `dealerName: string`, `lineItems: ComputedLine[]`, `subtotal: number`, `total: number`. New `display` derived value: `isReadOnly && initial` → persisted snapshot; else falls back to `computed.out.*` for draft. Tax row renders `fmtMoney(initial.tax)` instead of the spin-button input when read-only. Dealer static label reads `initial.dealerName` directly.
- `[id]/page.tsx` — threads `dealerName`, `lineItems`, `subtotal`/`tax`/`total` (Number-coerced) through to `initial`.

### Pass 2 — Verify

> Medium 1, archived dealer rendering, is closed (`initial.dealerName` rendered directly).
> Medium 2, read-only snapshot drift, is closed (persisted `lineItems`/`subtotal`/`tax`/`total` rendered when `isReadOnly`).
> Non-read-only invalid-input error path preserved (the `{ display.ok ? <table> : <error> }` discriminant still works because the three-arm `display` collapse keeps the error shape on the unhappy branch).
> Create-mode dealer Combobox still renders (no `initial` → `isEdit=false` → Combobox path).

**Pass-2 Low (deferred):**

1. **Malformed persisted `lineItems` could break read-only rendering** — `row.lineItems as unknown as ComputedLine[]` would silently let a v0-shape or non-array jsonb value through, and a renderer that reads `l.qty` could crash. Codex's own qualifier: "follows the repo's 'trust internal writes' convention" — the write path (`persistComputationPatch` → `computeQuote`) only ever emits canonical `ComputedLine[]`. Per CLAUDE.md → "Don't add error handling, fallbacks, or validation for scenarios that can't happen. Trust internal code and framework guarantees." Defer to a follow-up only if a malformed legacy row ever surfaces in prod.

## Coverage Caveats

- The single seeded quote is `status='draft'` against an active dealer — the read-only path (sent/accepted/declined) + archived-dealer hydration aren't exercisable by smoke without a fixture. Both are covered by the Codex pass-2 evidence walk + the Phase 5 plan note recommending a "non-draft fixture if dev DB tolerance allows" smoke.
- The composer's `setQuoteInputs` save path wasn't exercised end-to-end (would mutate the existing draft's totals). The TOCTOU re-classify path was confirmed by Codex pass-1 reading `actions.ts:315-326` + `legacy-result.ts:41` + the composer's `toast.error(result.error)`.
- `params.id` validation: `Number.isInteger(Number(idParam))` correctly catches non-integer strings (`'1.5'` → false), but accepts scientific notation (`'1e10'` → 10,000,000,000 integer). `loadQuote(1e10)` returns null → `notFound()` so the user-facing outcome is the same.
