### False positives Codex considered and ruled out

- **Migration apply-time blockers** — `drizzle/0017_tranquil_living_mummy.sql:1-5` uses plain `ALTER TABLE … DROP COLUMN` for only the five removed columns. The 0017 snapshot's `campaigns` indexes/FKs keep `audience_source_id` and `accepted_quote_id` only, with no index/FK on the five dropped columns — no dependency blocker.
- **Snapshot drift** — diffing `drizzle/meta/0016_snapshot.json` against `drizzle/meta/0017_snapshot.json` after removing only the 5 cols + snapshot id/prevId produced an exact match. No extra drift.
- **Journal ordering** — `_journal.json:117-129` monotonic across 0016 → 0017 (1779465600000 → 1779552000000).
- **Lurking code readers** — targeted greps for dynamic SQL/template access, bracket access, and bare fixture keys found no remaining `campaigns` readers for the 5 dropped names. Hits in `quotes.ts` / `lib/quotes/*` / `features/quotes/*` are quote/composer surfaces, not legacy campaign reads.
- **CSV/export compatibility** — `production/export/route.ts:8-21` and `reports/export/route.ts:28-41` headers do not include the 5 dropped commercial columns.
- **RLS/policy bindings** — `drizzle/0003_enable_rls.sql:96-108` defines campaigns policies in terms of `true` and `is_staff_member()`, not any dropped column.
