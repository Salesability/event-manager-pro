## Codex Adversarial Review

Verbatim from `codex:codex-rescue`:

**Critical (ship-blocker):** None.
**High (must fix this phase):** None.

**Medium (should fix this phase or note in plan)**

> Archive affordance is staff-visible but admin-only server-side: `src/features/dealers/dealers-columns.tsx:91` renders the archive button for every `/dealerships` viewer, while `src/features/schedule/actions.ts:266` requires `admin`. `/dealerships` itself is only layout-gated by staff access via `src/app/(app)/layout.tsx:7`, unlike People, which is page-gated admin-only at `src/app/(app)/admin/people/page.tsx:8`. This means staff/coach users can see and click a delete control that redirects/fails inside the server action. Suggested fix: either pass role/admin state and hide archive for non-admins, or widen `archiveDealer` if coaches/staff are intended to fully manage dealers.

**Note (Eval):** This is a **pre-existing** concern, not a Phase 2 regression. The original `list-actions.tsx:DealerRowActions` rendered the same `✕` button unconditionally to every staff viewer (verified by reading the pre-Phase-1 file content). Phase 2 preserves the affordance verbatim. Decision is whether to fold a fix into this phase, into Phase 3 (which rewrites the row actions anyway), or carry forward as a separate item.

**Low / nit**

> Active tracker still describes the old route decision: `docs/designs/CURRENT.md:16` still says `/lists` polish, and `:30` still says `/lists` → `/dealers` was open with working assumption "keep `/lists`". This is not live app code, but it will mislead the next agent reading the active design pointer.

> Phase 3 plan text still names the old app path: `docs/designs/0027-lists-to-dealers/plan.md:100` says `app/(app)/lists/dealer-form.tsx`, while the checklist below correctly references `src/app/(app)/dealerships/dealer-form.tsx` at `:102`.
