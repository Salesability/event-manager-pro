## Codex Adversarial Review

### Critical
None found.

### High
None found.

### Medium
1. **Smoke cleanup can delete real campaigns with matching notes**, `scripts/calendar-clamp-smoke.ts:53`. The script deletes every row where `campaigns.notes === 'CLAMP-SMOKE-2026-05-03'` before both `insert` and `cleanup`; a real user entering that exact note would lose data. Use a generated `publicId` prefix plus exact fixture dates, or store fixture IDs in a local temp file and delete by ID.
2. **Smoke insert can attach fixtures to an archived dealer**, `scripts/calendar-clamp-smoke.ts:63`. Normal app reads active dealers with `isNull(dealers.archivedAt)` in `queries.ts:169`, and campaign creation rejects archived dealers in `actions.ts:395`. Add `where(isNull(dealers.archivedAt))`; if no active dealer exists, fail before mutating.
3. **Rendered cross-month events can disagree with the "This Month" stat**, `calendar-view.tsx:399`. The new clamp renders events whose range overlaps the grid via `calendar-view.tsx:223`, but stats count only campaigns whose `startDate` is in the current month. Repro: `Apr 28 → May 4` renders in May, but May's stat excludes it. Count overlap with the visible month if the stat is meant to reflect visible campaigns.

### Low
1. **`clampToGrid`'s exported API silently assumes a 42-cell grid**, `calendar-grid.ts:20`. In production `CalendarView` builds 42 cells, but the helper type accepts any `Grid`; a shorter grid with `endDate > lastDate` returns `ei: 41`, which may be invalid for that grid. Either encode the 42-cell invariant in the helper name/type/tests, or derive the last index from `grid.indexOf[grid.lastDate]`.
2. **`getCoachColor` makes the ribbon draw callback unstable**, `calendar-view.tsx:122` (pre-existing). Recreated every render and included in `drawRibbons` deps at `calendar-view.tsx:375`, so the callback memoization is mostly ineffective on a hot DOM-drawing path. Wrap `getCoachColor` in `useCallback([coachColorMap])` or inline the lookup inside the callback.
3. **Smoke inserts are not transactional**, `scripts/calendar-clamp-smoke.ts:73`. If the first insert succeeds and the second fails, the script leaves a partial fixture set until manual cleanup.
4. **Smoke fixtures default to `draft`**, `scripts/calendar-clamp-smoke.ts:74`. Real `createCampaign` writes `status: 'booked'` at `actions.ts:402`, while the schema default is `draft` at `campaigns.ts:57`. Visual smoke still renders because `loadCampaigns` does not filter status, but the fixture is less representative than intended.

### Looks Suspicious But Actually Fine (verbatim, dropped duplicates)
- Per-row overlap uses clamped indices (`calendar-view.tsx:243`) — preserves the visible-row overlap test.
- `drawRibbons` reuses the packer's `_si`/`_ei` (`calendar-view.tsx:320`) — prevents the two passes from drifting after clamping.
- `cellIndexMap` key format matches the helper's lex assumption (`calendar-view.tsx:79`) — grid dates are zero-padded `YYYY-MM-DD`, validators enforce, DB uses `date`.
- DST/leap-year concerns do not hit the clamp helper (`calendar-grid.ts:17`) — string compares only.
- Hook deps are not stale after the refactor (`calendar-view.tsx:268`, `:375`).
