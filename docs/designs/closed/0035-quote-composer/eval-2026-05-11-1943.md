## Codex Adversarial Review

### Critical
None.

### High
None.

### Medium

1. **Archived service codes are hidden but still permanently block recreation.** `loadServiceItems()` hides archived rows, `archiveServiceItem()` only sets `archivedAt`, and `createServiceItem()` inserts a new row against a global unique `code`. An admin who archives `base-event` and tries to add it back gets "That code is already in use" with no admin UI recovery path. Files: `src/features/services/queries.ts:20`, `src/features/services/actions.ts:113`, `drizzle/0012_round_skrulls.sql:13`. Confidence: observed.

2. **Money parsing silently rounds through JS `Number`.** `parseNumericMoney()` does `Number(raw).toFixed(2)`, so `unitPrice=2.675` stores `2.67` and `0.105` stores `0.10`. Browser `step="0.01"` is not a server-side guard; direct Server Action calls trigger this. File: `src/features/services/actions.ts:34`. Confidence: observed.

3. **Numeric/integer overflow accepted by validation, only fails at the DB.** `unitPrice=100000000` passes `Number.isFinite(n) && n >= 0` but exceeds `numeric(10,2)`; `sortOrder=2147483648` passes `Number.isInteger` but exceeds Postgres `integer`. Files: `src/features/services/actions.ts:34`, `:52`, `src/lib/db/schema/service-items.ts:22`. Confidence: observed.

4. **New service Server Actions not represented in the executable action-gate matrix.** The matrix imports people/schedule/email/quotes actions and lists existing lookup actions, but `createServiceItem` / `updateServiceItem` / `archiveServiceItem` are absent — a future weakening of one service action's gate would not be caught. Files: `src/features/__tests__/action-gate-matrix.ts:13`, `:161`, `src/features/services/actions.ts:104`. Confidence: observed.

### Low

1. **Duplicate-code handling depends on driver error-message text.** Race behavior is fine today; the sharp edge is that `isDuplicateCodeError()` matches message substrings rather than SQLSTATE/constraint name — a driver/message-format change could turn a duplicate race into a generic server error. Files: `src/features/services/actions.ts:94`, `actions.test.ts:243`. Confidence: hypothesis.

2. **Range/non-range invariant enforced only in the Server Action, not the table.** The action clears wrong-side price fields, but no CHECK constraint enforces "range requires min+max & no unit_price" or "non-range forbids min/max." A future seed/backfill/admin-SQL path can insert inconsistent rows. Files: `drizzle/0012_round_skrulls.sql:2`, `src/features/services/actions.ts:74`. Confidence: likely.

### False-positives considered (dismissed)
Capability-gate bypass · RLS policy mismatch · Admin UI XSS · range/non-range wrong-field coexistence in the action path · seed catalog shape · journal monotonicity.
