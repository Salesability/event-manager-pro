## Codex Adversarial Review

### Critical / High
None.

### Medium

1. **`updateDealer` can write through an archive race and can clobber a newer status** — `src/features/schedule/actions.ts:188,195,202`. Existence check uses `archivedAt IS NULL` but the UPDATE only uses `id`. A concurrent `archiveDealer` between those statements would still let this action rewrite the archived row. The fallback status read is also stale by write time: a caller that omits `status` can overwrite a concurrent `prospect → active` transition. Fix: guarded UPDATE with `archivedAt IS NULL` + RETURNING; treat `status`/`acquiredVia` as patches (omit when absent).

2. **Archived rows show an Edit action that fails on save** — `src/features/dealers/dealers-columns.tsx:123`, `src/features/schedule/actions.ts:188`. Phase 2 loads archived dealers into /dealerships; archive button is hidden but Edit is rendered unconditionally. The save path rejects archived dealers, leaving the Archived pill with rows that invite editing but fail on submit. Fix: hide Edit on archived rows (mirror the archive-button conditional).

### Low

1. **`acquiredVia` not preserved when omitted from update form data** — `src/features/schedule/actions.ts:68,197`. Unlike `status`, no existing-row fallback. Current `DealerForm` always submits the field, so normal UI edits are fine; older/partial callers that omit `acquiredVia` would clear it.

2. **Backfill marks every existing dealer active with no discriminator** — `drizzle/0015_cultured_tinkerer.sql:2`. Migration adds `status DEFAULT 'active' NOT NULL`. Likely consistent with the v1 manual-back-office funnel (no pre-quote dealers in the dev/prod set), but if any rows are actually pre-sale prospects they need a manual correction script.

3. **`DealerForm.defaultStatus` is a create-flow prop but not enforced as create-only** — `src/features/dealers/dealer-form.tsx:59,175`. Documented as inline-composer use, accepted in any mode. If a future edit caller passes `defaultStatus='prospect'`, the visible select disappears and the hidden input overwrites the existing dealer status.

### False Positives Considered (dismissed)
- `convertProspectToActive` audit race
- Idempotent no-op behavior
- Accidental NULL status on normal update
- Audit enum ordering
- RLS coverage for new dealer columns
- Capability matrix drift
- `loadDealers()` excluding prospects
