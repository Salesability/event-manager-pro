## Carry-forwards (from earlier phases)

These are explicit non-blockers from prior eval reports, restated here as the chunk-close summary:

1. **Audit-emit gap for dealer role grants** (Codex Medium from Phase 4). `createDealer` / `updateDealer` / Phase 2 backfill don't emit `user.role_changed` audit rows. Wiring this cleanly needs a SELECT-then-conditional helper to distinguish granted/un-archived/no-op outcomes — warrants a small dedicated audit-emit chunk for dealer paths.
2. **Plan drift: `dealer + admin/coach` hybrid** (Codex Low from Phase 3). Allowed silently today; plan's open question envisioned an "are you sure?" confirm. Acceptable for v1 since staff-app roles are explicitly distinguished from `dealer` at every authz layer (`STAFF_APP_ROLES`, `is_staff_member()` whitelist).
3. **Production data triage** (Codex consideration from Phase 5). Re-run `scripts/backfill-dealer-role.ts --dry-run` against prod before this chunk reaches there — dev had zero truly-orphan contacts but prod is unknown.
4. **Schedule-actions test fixture** (recurring deferral). Several phases punted on a vitest fixture for `src/features/schedule/actions.ts`. Worth investing once a dedicated tests-layer pass surfaces.
5. **Real form-submission test** (Phase 3 finding). The `4a4afbd refactor(forms)` regression that silently broke role-submission for hours wasn't caught by any existing test because all people-admin tests construct FormData manually. A Playwright/RTL test that submits the dialog would have caught it. Defer until destructive-flow concerns can be sidestepped (fixture-and-cleanup script).
