## Codex Adversarial Review

Two Medium findings, both quoted verbatim from the Codex run:

### M1 — aria-invalid vs data-invalid (FIXED IN-CYCLE at `f6e9aeb`)

> Validation controls set `aria-invalid`, but Catalyst styling listens for `data-invalid`.
> Citation: `src/components/catalyst/input.tsx:82`, `src/features/dealers/dealer-form.tsx:123`
> Impact: invalid inputs keep screen-reader state and red error text, but the red Catalyst border state does not render across touched forms using `aria-invalid`.
> Suggested fix: pass Headless UI's `invalid={...}` prop into `Input`/`Textarea` callsites, or have the Catalyst wrappers translate `aria-invalid` into `invalid`/`data-invalid`.

**In-cycle resolution:** extended Catalyst `input.tsx` and `textarea.tsx` to also style on `aria-[invalid=true]:border-red-500` alongside the existing `data-invalid:border-red-500`. This makes the existing `aria-invalid={!!errors.x}` on every callsite paint the red invalid border without touching the 6+ form files. Catalyst sources at `input.tsx` + `textarea.tsx` are now deliberately edited (alongside the `brand` color entries added to `button.tsx`/`badge.tsx` in Phase 2).

### M2 — Custom Tabs missing keyboard/ARIA contract (PARKED as 0049 follow-up (a))

> Custom Tabs omit the tab keyboard and ARIA wiring that Base UI supplied.
> Citation: `src/components/catalyst/tabs.tsx:38`, `src/components/catalyst/tabs.tsx:60`, `src/components/catalyst/tabs.tsx:88`
> Impact: `/reports` tabs only change on click; keyboard users do not get Arrow/Home/End tab navigation, and panels are not connected to tabs via `id`/`aria-controls`/`aria-labelledby`.
> Suggested fix: replace the custom context implementation with Headless UI `TabGroup`, or implement the full WAI-ARIA tabs keyboard and relationship contract.

**Parked rationale:** the regression is real (lost from the Base UI Tabs) but localized to one route (`/reports` only — 1 callsite via `reports-tabs.tsx`). Out-of-scope for an in-cycle fix because rebuilding on Headless UI `<TabGroup>` is structurally distinct from the existing context-based shape (TabGroup uses index-based `selectedIndex`/`onChange`, our Tabs uses value-based `value`/`onValueChange` — the wrapper has to bridge). Worth its own small chunk.
