## Codex Adversarial Review

### Medium

- **Lossy bigint reads** — `src/lib/db/schema/_columns.ts:21` (mode: 'number'). `loadAdminPeople` inherits the project-wide convention of `bigint(..., { mode: 'number' })` for domain IDs. `src/features/people/queries.ts:51` reads `contacts.id` and dealer ids as numbers, so values past 2^53 round before `inArray` / Map stitching. Same caveat as the rest of the schema — `db-conventions` skill explicitly accepts this for the project's scale.

### Low

- **Drizzle mock ignores predicates** — `src/features/people/queries.test.ts:13`. `.where()` and `.innerJoin()` just return queued results. Production code that drops an `archived_at IS NULL` filter would still pass. Same shape as the 0018 test mock — known follow-up captured in `CURRENT.md` Parked.
- **Dealer-link ordering only by `dealerId`** (second-order) — `src/features/people/queries.ts:110`. Multiple links for the same dealer/contact return in unspecified role order. Future UI snapshots could be flaky.

### Nit

- **`DealerContactRole` is hand-duplicated** — `src/features/people/queries.ts:14`. The schema enum lives in `src/lib/db/schema/dealer-contacts.ts:16`. If the enum gains a value, this type silently drifts. Could be inferred via Drizzle's `$inferSelect` or imported from a shared types file.

### Confirmed-OK (false-positive reduction)

- Bulk reads, no N+1: roles / dealer links / identifiers all use `inArray(contactIds, ...)` with in-memory map stitching.
- `auth.admin.listUsers` skipped when no contact has a `user_id` (matches `loadAdminUsers` shape).
- `hasDealerContact` (the 0018 LIMIT-1 gate probe) is **not** re-implemented here — different concern, different shape.
- Dealer + dealer-link archived-at filters both present (matches `loadDealers` / `fetchPrimaryDealerContacts` convention).
- Identifier de-dup delegated to the `contact_identifiers_contact_kind_primary_unique` partial index — no client-side dedupe needed.
- Missing auth user (linked but `admin.listUsers` doesn't return it) degrades gracefully → `hasAppAccess: false`, email falls back through identifier or null.
- `'server-only'` import is first; blocks Client Component import.
- No SQL injection surface: Drizzle predicate builders only, no raw `sql\`\`` interpolation.
