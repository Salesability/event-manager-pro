### Stale-comment cleanups landed in-eval

**Risk 1 — Test comment misleading.** `src/lib/actions/action-client.test.ts:107-109` said "capability admin-shortcut bypasses the membership lookup" — but `assertCan` loads membership unconditionally before invoking `can()`. The admin shortcut admits even with empty roles, but the membership load still happens. Behavior is correct; comment was lying about the implementation.
**Resolution:** reworded to "assertCan loads membership unconditionally; the admin shortcut inside can() admits via app_metadata.role even when the role list is empty."

**Risk 7b — Audit doc comment stale.** `src/features/audit/actions.ts:22-23` said "every caller is downstream of `requireRole(...)`." After Phase 1+2+3, every caller is downstream of a capability gate (`assertCan` / `capabilityClient`).
**Resolution:** updated to "every caller is downstream of a capability gate (`assertCan` / `capabilityClient`) that already redirected an unauthed request."
