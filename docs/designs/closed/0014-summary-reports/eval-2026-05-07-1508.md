## Codex Adversarial Review

**Critical / High:** none.

**Medium (2 found, both fixed in-eval):**

1. **CSV mitigation regex too narrow (FIXED).** Original `/^[=+\-@\t\r]/` only saw the leading char; Excel/Sheets trim leading whitespace before parsing, so `\n=HYPERLINK(...)`, ` =SUM(A1)`, NBSP+`=`, BOM+`=` all bypassed. Resolution: changed to `/^\s*[=+\-@]/` — modern JS `\s` already covers NBSP and ZWNBSP/BOM, so a single `\s*` covers every leading-whitespace bypass without enumerating invisible-char codepoints. Test extended with the four bypass patterns (`\n=`, ` =`, NBSP+`=`, BOM+`=`) plus a regression case showing bare-`\t` non-formula values are NO LONGER prefixed (the previous regex was over-eager).
2. **Print-pagination trap (FIXED).** Hiding the pagination footer with `print:hidden` left users on page 2 silently printing only page-2 rows. Resolution: `<DataTable>` registers `beforeprint` / `afterprint` listeners and toggles between `getRowModel().rows` (paginated) and `getFilteredRowModel().rows` (all filtered) so the printed output respects the user's search/filter selection but ignores the 25-per-page slicing. Sort + filter state survive because they're upstream of the pagination row model.

**Low (1, deferred):**

- `csvResponse(filename, …)` is safe today because filenames come from internal whitelists + ISO date strings. If the helper ever takes user-controlled filenames, it'll need filename sanitization or RFC 5987 (`Content-Disposition: attachment; filename*=UTF-8''…`). Captured in plan as a future concern; not actionable now.

### Codex "Not An Issue" (false-positive reduction)

- `?tab=` parser is whitelist-only — no traversal, no SSRF, no open redirect.
- `requireRole(['admin','coach'])` matches the page gate; route handler doesn't escalate access.
- `Cache-Control: no-store` appropriate for staff/admin CSVs; Next route handlers aren't cached by default but the header hardens against downstream caches.
- Production export refactor preserves wire format: old `c.qtyRecords ?? ''` + `String(v)` → new `c.qtyRecords != null ? String(c.qtyRecords) : ''`. Both produce numeric string for values, empty string for null.
- Quote-doubling does not create a formula bypass; raw value starting with `"` is parsed as text after CSV unquoting, not as a formula.
- Coach `groupKey: null` → CSV cell `"Unassigned"` (the row's `groupLabel` field). Stable.
