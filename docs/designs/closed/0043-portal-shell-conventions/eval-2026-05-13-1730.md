### Consider (Codex medium — parked)

1. **0043 follow-up (a) — composer-actions lift.** Sticky `<PageHeader>` on `/quotes/new` + `/quotes/[id]` adds chrome but doesn't host the composer actions (Save / Preview / Send / Re-send / Close). Lift those actions out of `quote-composer.tsx:594` into the `<PageHeader actions>` slot (the chunk-spirit fix), or revert `sticky` on those routes if the lift can't ship soon. Already on `CURRENT.md`'s sequence as the natural companion to 0046's composer work — parked here pinning the eval-finding as evidence.

2. **0043 follow-up (b) — row-action sweep into non-`*-columns.tsx` surfaces.** Three callsites still render hand-rolled per-row actions outside the lint rule's path scope: the nested Quotes table on `/dealerships/[id]:225`, `services-admin.tsx:132`, `availability-admin.tsx:191`. Fix shape: either port to `<RowActions>` (preferred — same vocabulary win as the rest of the chunk) or widen the lint rule's glob to cover the file shapes. The chunk's spirit applies; the boundary is where energy ran out.

3. **0043 follow-up (c) — status-pill sweep into calendar/people.** Calendar `event-detail.tsx` still renders campaign status via local `statusBadgeClass()`; `people-columns.tsx` renders lifecycle status with a hand-rolled span. Fix shape: add `<CampaignStatusBadge status>` (raw enum variant) + `<PersonLifecycleBadge>`, replace the inline chrome.

### Consider (Codex low — parked)

4. **0043 follow-up (d) — retire `STATUS_PILL_CLS`.** No swept callsite uses the map; deleting it (and its test block) closes the back-door for inline status-pill regressions.

5. **0043 follow-up (e) — add `Quote` to the lint FORBIDDEN set.** Either derive `FORBIDDEN` from `ROW_ACTION_LABELS` directly (best — single source) or add `'Quote'` to the static set so the dealer-row `Quote` label can't drift back to inline literals.
