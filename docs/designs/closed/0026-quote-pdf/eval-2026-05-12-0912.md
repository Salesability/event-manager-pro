## Codex Adversarial Review

### High

- **[src/features/quotes/actions.ts:496]** Send can persist a PDF from a stale draft snapshot — `sendQuote` preloads line items/totals, renders/uploads, then only guards the final update on `status='draft'` at [src/features/quotes/actions.ts:572], while draft setters can still update the same draft at [src/features/quotes/actions.ts:299], [src/features/quotes/actions.ts:356], and [src/features/quotes/actions.ts:407]. A concurrent edit between preload and final update can make the row's final commercial terms differ from the uploaded PDF; include `updatedAt`/snapshot hash in the guarded update, or transition to a locked/intermediate "sending" state before rendering.
- **[src/lib/storage/gcs.ts:54]** Concurrent sends can overwrite the winning PDF object — `putObject` uses plain `file.save` with no generation/If-None-Match precondition, and all senders write the fixed key from [src/features/quotes/actions.ts:435] before race classification at [src/features/quotes/actions.ts:575]. A losing sender can overwrite `quotes/{quoteId}/1.pdf` after another caller has already sent the quote; use a GCS generation precondition such as `ifGenerationMatch: 0`, or write attempt-unique keys and only persist/promote the winner's key.

### Medium

- **[src/features/quotes/actions.ts:443]** Malformed persisted `lineItems` are silently converted into client-visible blank/zero PDF rows — `toRendererLines` returns `[]` for non-arrays and coerces missing fields with `String(... ?? '')` / `Number(... ?? 0)` at [src/features/quotes/actions.ts:444]. Because `line_items` is unconstrained jsonb at [src/lib/db/schema/quotes.ts:76], corruption or bad migration data can produce a sent PDF that masks missing labels/prices; validate the JSON shape and finite numbers fail-closed before rendering.
- **[src/lib/pdf/render-quote.ts:32]** `MAX_LINE_ITEMS = 25` does not actually leave room for totals and terms in the current single-page layout — each row consumes 16pt at [src/lib/pdf/render-quote.ts:231], then totals and terms continue at [src/lib/pdf/render-quote.ts:251] and [src/lib/pdf/render-quote.ts:273]. The test only asserts one page at [src/lib/pdf/render-quote.test.ts:51], not that content stays on-page; lower the cap to the measured safe maximum or add y-position assertions/multipage rendering.
- **[src/lib/pdf/render-quote.ts:191]** Free-form client text is drawn without wrapping or Unicode font support — dealer name/address lines from [src/features/quotes/actions.ts:532] and [src/features/quotes/actions.ts:533] are passed directly to Helvetica drawing at [src/lib/pdf/render-quote.ts:191] and [src/lib/pdf/render-quote.ts:195]. Long address lines can overflow the page, and non-WinAnsi/RTL characters can make rendering fail; wrap/truncate bounded fields and embed a Unicode-capable font or validate unsupported characters before send.
- **[src/features/quotes/actions.ts:471]** `sendQuote` is gated by `quote:edit`, which admits coaches at [src/lib/auth/capabilities.ts:77], but the action is now an irreversible lifecycle transition with external-document side effects rather than just draft editing. The matrix has no `quote:send` capability at [src/lib/auth/capabilities.ts:17], while outbound communications are separately admin-only in [docs/wiki/auth.md:108]; add an explicit `quote:send` capability with the intended admit set and gate the send UI/action on it.

### Low

- **[src/features/quotes/actions.ts:538]** The PDF hardcodes every quote's event as `Sales Event` even when no actual event name is known. This is not a data-integrity break, but it is user-facing and can confuse clients; make `eventName` nullable and omit the Event block until quote/campaign linking provides a real value.
- **[src/features/quotes/actions.test.ts:339]** The send tests cover update-miss classification but not the destructive storage ordering behind it. Add adversarial tests for "concurrent edit between preload and update" and "losing concurrent send uploads after the winner" so the stale-PDF/GCS overwrite fixes stay pinned.

### Should Fix (Codex High/Medium)

Routing to `codex:codex-rescue` self-resolve loop (pass 1 of 2 per build-skill budget):

1. **High 1** — `sendQuote` stale-snapshot race. Add `updatedAt` snapshot capture at preload + include in guarded UPDATE `WHERE`-clause so concurrent setters between preload and UPDATE force a re-classify.
2. **High 2** — GCS overwrite race. Add `ifGenerationMatch: 0` precondition to `putObject` so concurrent senders writing the same key get a generation-mismatch error from GCS.
3. **Medium 1** — `toRendererLines` shape validation. Fail closed on non-finite numbers / missing required fields rather than silently coercing to blank/zero.
4. **Medium 2** — `MAX_LINE_ITEMS` cap. Measure the actual safe max against the layout's totals + terms footprint; lower the constant and update the test.
5. **Medium 3** — Free-form client text. Truncate dealer name + address lines via `font.widthOfTextAtSize` measurement; strip non-WinAnsi chars.
6. **Low 2** — Test gaps. Add adversarial cases for the High-1 and High-2 fixes.
