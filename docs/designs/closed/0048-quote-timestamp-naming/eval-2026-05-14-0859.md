## Codex Adversarial Review

Codex returned the following findings verbatim:

### Low

**1. DST fall-back collision in `quoteDisplayName`** — `src/features/quotes/display-name.ts:18`

DST fall-back creates identical display names for two distinct instants an hour apart, so the timestamp is not a stable unique user-visible identity during the repeated local hour.

Minimal repro: `quoteDisplayName(new Date('2026-11-01T05:30:00Z'))` and `quoteDisplayName(new Date('2026-11-01T06:30:00Z'))` both produce `quote-20261101-0130` in `America/Toronto`. This also means `quoteDownloadFilename` collides for those two quotes.

**Disposition:** Parked as 0048 follow-up (a) — see "Action Required → Consider" below.

**2. Hypothetical null mismatch in MSA dialog** — `src/features/msa/msa-create-dialog.tsx:124`

Hypothesis: the null fallback still leaks the database id as a user-visible quote identity if `firstDraftQuoteId` is non-null but `firstDraftQuoteCreatedAt` is null. The current server caller passes both values from the same `firstDraftQuote` object, so this should not happen in the normal path. The prop type allows the mismatch, though, and the fallback violates the chunk goal that row ids are no longer user-visible.

**Disposition:** Fixed in-cycle at `bc4d423` — collapsed the two parallel nullable props (`firstDraftQuoteId` + `firstDraftQuoteCreatedAt`) into a single discriminated object `firstDraftQuote: { id: number; createdAt: Date } | null`. The leaky `quote ${props.firstDraftQuoteId}` fallback branch is gone by construction. `msa-panel.tsx` + `dealerships/[id]/page.tsx` callers updated. Post-fix tsc clean / 802/804 tests PASS / smoke re-verified the dialog renders the timestamp form.

### Consider (Codex low — parked)

1. **0048 follow-up (a): DST fall-back collision in `quoteDisplayName`** (Codex Low — parked). During the autumn DST transition (the repeated local hour, roughly 01:00–02:00 local), two distinct UTC instants produce identical `quote-YYYYMMDD-HHmm` display names + identical `saledayevents-quote-<ts>.pdf` filenames. Server-side persistence (GCS path `quotes/{quoteId}/{revision}.pdf` — id-keyed) is unaffected; the collision is presentation-only and lasts one hour per year. Fix shape options: (i) include seconds in the display name (`quote-YYYYMMDD-HHmmss`), trading shorter names for total uniqueness; (ii) include a short suffix derived from the row id (`quote-YYYYMMDD-HHmm-<id-tail>`); (iii) emit the local-time offset to disambiguate (`quote-YYYYMMDD-HHmm-0400` vs `-0500`). Un-park trigger: a customer/coach reports two quotes sharing a display name in the November fall-back window, OR the next chunk that touches quote-identity display.
