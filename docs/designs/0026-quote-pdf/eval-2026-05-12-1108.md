## Codex Adversarial Review

### High

- **[`src/app/quote/[token]/[action]/route.ts:42,73-76`]** Mutates quote state from a GET, and the email template sends those URLs as ordinary one-click links; concrete reason: email scanners/prefetchers can accept or decline a quote without an intentional client confirmation.
- **[`src/features/quotes/actions.ts:605-620,639-647,662-676,679-684`]** Commits `status: 'sent'` before upload/email/audit, and treats any later `sent` retry as success; concrete reason: a PDF upload or Resend failure leaves a sent quote that was not delivered and cannot be retried through this action.

### Medium

- **[`src/features/quotes/actions.ts:557-566`]** Loads the dealer by id without `isNull(dealers.archivedAt)`, while dealers are explicitly archivable; concrete reason: a draft whose dealer was archived after creation can still be sent publicly.
- **[`src/app/quote/[token]/[action]/route.ts:73-76`]** Accepts by only calling `markQuoteAccepted`; concrete reason: the dealer lifecycle note says accepted quotes should automatically activate prospect dealers (`convertProspectToActive` at `src/features/schedule/actions.ts:358-360`), but this public accept path never performs that promotion.

### Low

- **[`src/lib/rate-limit.ts:32,54-56,72-74`]** Stores per-key buckets without pruning expired keys; concrete reason: varied IP/header keys can grow the in-memory map indefinitely on the public route.

## Cross-Reference

The two Highs reinforced each other: H1 made the public-link UX unsafe in the presence of corporate email scanners; H2 made the staff-side recovery from email-delivery failure impossible through the existing surface. Together they made the original Phase 4 design unshippable for production B2B (Outlook/Microsoft-365 recipients) without a more substantial reshape. Reshape resolved H1 by structural decision (no public surface); H2 deferred to Phase 5 (per the pre-existing rescue-2 follow-up note).

## Action Required

### Must Fix (blockers)

None — clean PASS post-reshape.

### Resolved by reshape

- **H1** (email-scanner auto-accept) — cleared by dropping the public accept/decline flow. Email now contains the PDF + "reply or call your coach" message; no in-email actuating links. The customer phones or replies; the coach flips status via `acceptQuote` / `declineQuote` staff Server Actions.
- **M1** (archive-aware dealer join) — fixed inline: `sendQuote` now joins `dealers` with `AND archived_at IS NULL`, returning `Dealer not found or archived.` on miss.
- **M2** (prospect → active promotion) — fixed inline: `acceptQuote` runs the same atomic guarded UPDATE that `convertProspectToActive` uses, emitting `dealer.activated` with payload `{ from: 'prospect', via: 'quote.accepted' }`.
- **L** (rate-limit memory growth) — moot after `src/lib/rate-limit.ts` was deleted.

### Should Fix (Phase 5 follow-up)

- **H2** (degraded-state lockout) — already in the Phase 5 checklist; needs `emailedAt` timestamp on `quotes` so `sendQuote` retry can distinguish "row sent but email failed" from "row truly sent." Schema change + retry logic.

### Consider

- `quote:send` separate from `quote:edit` (Codex Medium 4 from Phase 3 eval) — still parked. The Phase 4 reshape doesn't change the gate decision.

## Coverage Caveats

- **No real-token end-to-end smoke** — the dev DB has zero `quotes` rows; building a fixture to test the actual send flow (Resend send + GCS upload) would touch real third-party APIs and email recipients. The 58 vitest cases in `src/features/quotes/actions.test.ts` cover the action surface; live verification is a Phase 5 dev-smoke item.
- **Codex pass-2 not run** — the user-directed reshape was a product-shape decision that wouldn't have benefitted from another adversarial pass (the dropped surface no longer exists to review). The orchestrator's inline M1 + M2 fixes are small (one-line dealer guard, one prospect-activation UPDATE) and inspected against the existing `convertProspectToActive` pattern.
- **Parallel-mode pool exhaustion** — when `pnpm test` runs at default concurrency, `tests/integration/rls.test.ts` flakes because the Supabase session pooler caps at 15 clients. Run with `--no-file-parallelism` for a stable signal. Carry-forward to fix via separate-project test config.
