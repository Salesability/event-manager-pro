## Codex Adversarial Review

**Critical:** None.

**High**

1. **Unbounded signed URL TTL** — `signedUrl(ttlSeconds)` (`gcs.ts`) accepts any value and passes it into `expires` directly. A caller bug can mint arbitrarily long-lived links to customer documents instead of being capped at a known policy boundary.
2. **Single-page renderer silently overflows with many line items** — `renderQuotePdf` creates exactly one US-Letter page and decrements `y` past page bottom on long inputs. The plan acknowledges the hard cap as a Phase-3 carry-forward; not blocking Phase 1, but flag persists.

**Medium**

3. **GCS adapter blocks ADC/workload identity unless `GCS_PROJECT_ID` is set** — `client()` hard-fails if `process.env.GCS_PROJECT_ID` is absent, which prevents the normal ADC path from inferring project ID. The `.env.example` comment promises workload identity works without inline JSON; the project-id check breaks that promise.
4. **Object keys are unsanitized across put/get/signed URL paths** — raw keys flow into `.file(...)`. Future callers can write leading-slash, `../`, newline, or wrong-prefix objects unless every caller validates. Foundation contract should enforce this once.
5. **User-controlled text drawn without width or character safety** — Bill-To, address, event, and line-item descriptions can collide with numeric columns or run off-page; unsupported glyphs in pdf-lib standard fonts can fail the whole render.
6. **Currency rendering accepts non-finite and oversized numbers** — `formatCurrency(NaN | Infinity | huge)` produces invalid-looking PDFs or negative x-coordinates instead of a controlled error.

**Low**

7. **`.env.example` says GCS stores branded PDF templates** — comment conflicts with the locked Phase-1 decision ("GCS holds rendered output only"). Could send future implementers back toward the abandoned template-fill model.
8. **Logo dependency hard-fails all rendering when asset missing** — `public/saledayevents-logo.jpg` rename or missing file blocks every quote send via the generic catch. No anchored test for this failure mode.

**False positives (confirmed not issues)**

- `import 'server-only'` is present on both adapters; vitest mock at `src/lib/pdf/render-quote.test.ts` is appropriate and does not weaken the runtime boundary.
- GCS error shape (`{ ok: true, ... } | { error: string }`) matches the `src/lib/email/send.ts` precedent.
- Inline JSON credential parsing — guards `if (process.env.GCS_CREDENTIALS_JSON)`, falls through to ADC otherwise.
- Module-level cached `Storage` client mirrors the cached-client shape in `email/send.ts`; no per-request data stored.

## Action Required

### Must Fix (blockers)

None. Lint blocker resolved in-eval.

### Should Fix (Codex high/medium)

1. **`.env.example` doc fix** (Codex Low #7) — line says "branded PDF templates and rendered Quote/Contract/Invoice PDFs"; correct to rendered-only per the locked decision. One-line fix worth bundling now.
2. **Cap signed URL TTL** (Codex High #1) — add a max-TTL clamp (e.g. 7 days) inside `signedUrl` so callers can't accidentally mint long-lived links. Cheap to add now; expensive if a feature ships first that abuses it.
3. **Drop the mandatory `GCS_PROJECT_ID` check** (Codex Medium #3) — let `Storage` infer projectId from ADC when `GCS_CREDENTIALS_JSON` is unset. Matches the `.env.example` comment's contract.

### Consider (Codex high/medium/low — Phase 3 carry-forward)

1. **Single-page overflow** (Codex High #2) — already in the plan as Phase 3 carry-forward; verify on first 50-line-item smoke.
2. **Key sanitization in `gcs.ts`** (Codex Medium #4) — defer to Phase 2/3 when callers materialize; foundation has no caller yet.
3. **Text overflow + glyph safety** (Codex Medium #5) — Phase 3, when real quote data lands.
4. **Currency input validation** (Codex Medium #6) — Phase 3, when `sendQuote` assembles `QuoteData` from the row.
5. **Logo asset failure-mode test** (Codex Low #8) — add when `render-quote.test.ts` grows.
