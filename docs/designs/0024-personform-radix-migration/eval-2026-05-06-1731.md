## Codex Adversarial Review

**Critical/High/Medium/Low:** None.

**Non-Issues** (all 10 priorities confirmed):

1. Hidden-input wire format — Radix forwards `name`/`value` to a real `<input type="checkbox">`; `parseRolesField`'s `formData.getAll('roles')` consumes it correctly.
2. `onCheckedChange` semantics — `(c) => setAdmin(c === true)` correctly coerces the `boolean | 'indeterminate'` shape.
3. Label association — wrapper `<label>` still binds; Radix's button-as-checkbox responds to label-text clicks.
4. `hasAnyRole` guard — derived from React state; `setChecked` fires synchronously.
5. Edit-mode default — controlled `checked` prop respected on first render via Radix's `useControllableState`.
6. Tab order + Space-to-toggle — Radix renders `<button role="checkbox">`; native button Space activation works.
7. Destructive-edit confirm — `wantsAppAccess = admin || coach` derivation unchanged.
8. Unchecked checkbox omission — Radix renders a real `type="checkbox"` (not `type="hidden"`); native FormData omits unchecked entries.
9. Tailwind `data-[state=checked]` — Radix writes `data-state="checked"` when checked; selector targets the same element.
10. Multiple same-name checkboxes — three independent `Checkbox.Root`s render three independent hidden inputs; FormData aggregates them as repeated entries.

Codex's bottom line: ship.
