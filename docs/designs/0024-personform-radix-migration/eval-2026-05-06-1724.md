## Codex Adversarial Review

**Critical:** None.

**High:**

1. **Focus return regresses without `Dialog.Trigger`.** Radix returns focus to a `Dialog.Trigger` on close by default; our consumers open dialogs via controlled `open` props with the trigger button outside `Dialog.Root`. Without intervention, Esc / outside-click / Close button would have landed focus on `<body>` — keyboard users lose their place. **Status: fixed in-eval.** Added a `FocusContext` (`React.MutableRefObject<HTMLElement | null>`) populated by `Root` on `open` flip via `document.activeElement` capture; `Panel`'s `onCloseAutoFocus` reads the saved ref, calls `e.preventDefault()`, and re-focuses the captured element. Restores the previous-focused-button-after-close UX HUI provided by default.

**Medium:** None.

**Low:**

2. **`aria-describedby` console warnings for dialogs without `Dialog.Description`.** Radix wires `aria-describedby` automatically and warns to console (`console.warn`, not error) when no matching `Description` exists. **Status: deferred.** Per Codex's dig-deeper note, this is actually fine for our current consumers — `people-admin` Add/Edit dialogs both have descriptions; `orphan-auth-users` has one too; the `Block Date` dialog has one. If any call site is missing a description, the warning is dev-only and non-functional. Worth a sweep with `rg -n "<Dialog\.Panel>|<Dialog\.Description>"` if console noise is observed in dev.

**Explicitly cleared (not issues):**
- Portal placement: single `Dialog.Portal` inside `Root` is fine; no consumer uses a Trigger inside Root.
- `onOpenChange` ↔ `onClose` translation: matches the close-only API; future trigger-open support would need API expansion but no caller depends on `open=true` callback.
- Close transition cut by unmount: documented in the wrapper comment; acceptable for v1, tunable with `forceMount` + tailwindcss-animate later.
- Z-index layering: `z-40` on Backdrop, `z-50` on Panel wrapper preserved.
- `forwardRef` + prop spreading: composes correctly for current consumers.
- Tailwind `data-[state=closed]:*` selectors target the right element (Radix Overlay/Content set `data-state`).
- No unused-import lint warnings; HUI dependency intentionally retained for Phase 5 rip-out.
