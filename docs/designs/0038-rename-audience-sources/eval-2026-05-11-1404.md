## Codex Adversarial Review

### Critical
None.

### High
- **Phase 1 leaves the TypeScript tree unbuildable.** `src/lib/db/schema/index.ts:1-13` no longer exports `salesLeadSources`, while consumers still import it at `src/features/schedule/queries.ts:5-15` and `src/features/schedule/actions.ts:7-17`; they also read removed `campaigns.salesLeadSourceId` at `src/features/schedule/queries.ts:346,361,409,424`. → **Corroborated by tsc errors above.**

### Medium
- **RLS policy names still carry the old table name.** `drizzle/0003_enable_rls.sql:188-198` created `sales_lead_sources_service_role_all` and `sales_lead_sources_staff_all`; `drizzle/0007_thin_thor.sql` renames table/sequence/constraints/index but has no `ALTER POLICY` statements. Policies stay attached by OID, so behavior is correct — but pg_dump-visible policy names remain stale. Cosmetic; can fold an `ALTER POLICY ... RENAME TO` into 0007, or accept the drift.

### Low
- **`0007_thin_thor.sql` is not safely re-runnable.** A second direct apply hard-fails because the RENAME statements require pre-rename object names. Acceptable under normal Drizzle forward-only bookkeeping (the migrator tracks via `__drizzle_migrations.hash` so won't re-run).
- **Out-of-band Supabase objects can't be ruled out from repo alone.** Repo search found no `CREATE PUBLICATION` / `supabase_realtime` / `GRANT … sales_lead` / `COMMENT ON` entries, but Studio-created policies/grants/publications aren't represented in migrations. Need live `pg_catalog` inspection to confirm none exist. Low risk for a dev DB with no realtime usage yet.

### Not an issue (false-positive head-off)
- RLS predicates don't reference the old table or column (only `true` and `public.is_staff_member()`).
- `is_staff_member()` reads `contacts` + `team_member_roles`, not the renamed objects.
- No triggers, views, materialized views, or generated columns reference the renamed objects.
- No other FKs into `sales_lead_sources` exist.
- `0007_snapshot.json` has no remaining old-name tokens; FK `tableFrom/tableTo` correctly point at `audience_sources`.
- Journal ordering and `prevId` are internally consistent; `when` is monotonically increasing.
- Sequence ownership survives explicit sequence rename — identity dependency is preserved.
