## Codex Adversarial Review

**High** — `src/features/quotes/actions.ts:316` — campaign/dealer guard raceable: the campaign row was read without a lock while the dealer row is `FOR UPDATE`-locked. A concurrent `updateCampaign` (`src/features/schedule/actions.ts:939`) can repoint `campaigns.dealerId` between the read and the quote insert under READ COMMITTED, persisting a quote whose `dealer_id` no longer matches its `campaign_id` — bypassing the same-dealer invariant. **→ FIXED `de54a4e`: added `.for('update')` to the campaign select.**

**Medium** — `src/features/schedule/commercial-status.ts:86` — expired-but-`active` MSAs reported as active: the resolver selected `status IN ('active','pending')` without checking `expiresAt`, but `acceptQuote` (`actions.ts:1283`) only honors an active MSA when `expiresAt >= now()` (expiry sweep not built). A stale active row past expiry would mark an exposed event "Protected." **→ FIXED `de54a4e`: new pure `effectiveMsaStatus()` mirrors the accept gate (active only while unexpired; active-but-expired → `expired`); now selects `expiresAt`; +6 unit tests.**

**Low** — `src/app/(app)/calendar/calendar-view.tsx:673` — post-booking "Send MSA for signature" CTA rendered without the `admin:access` gate the event-detail CTA uses; a non-admin campaign-creator sees a dead-end nav to the admin-gated dealer page (no security hole — dealer page is server-gated — but a leaked/unavailable workflow). **→ FIXED `de54a4e`: wrapped in `<Can capability="admin:access">`.**

**Explicit non-issues (false-positive reduction):** empty-`inArray` guarded by the early return; commercial-status is 2 batch queries (no N+1); non-concurrent cross-dealer `campaignId` already rejected by the dealer-match guard; backfill `0047` idempotent (`IS NULL` only); RSC `Map`→plain-object serialization correct; missing map entries handled by optional chaining.
