## Codex Adversarial Review

### Pass 1 (initial, diff `3dc5f58..HEAD`)
- **High** — Cross-dealer overwrite: `setQuoteDealer` swaps a draft quote's `dealerId` without reconciling `campaignId`; the accept-time write could overwrite another dealer's campaign. **→ FIXED** `67655a7` (writer scopes the UPDATE to `campaigns.dealerId = quote.dealerId`).
- **High** — Idempotent re-accept regression: running on every confirmed-accepted call let a re-accept of an older quote regress the campaign. **→ FIXED** `67655a7` (call moved inside `if (result.transitioned)`).
- **Medium** — Integer overflow: a pathological quote (≥5 `base-event` lines at qty 1M) derives >int4; the campaign UPDATE fails after the quote already flipped accepted. **→ PARKED 0094-b** (adversarial-only; loud-failing; the composer never generates this).

### Pass 2 (re-review of the fix)
- **Prior Highs:** both **RESOLVED** (confirmed by Codex).
- **High** — Concurrent-accept race: two *different* `sent` quotes on the *same* campaign accepted near-simultaneously can leave the campaign reflecting the non-latest one. **→ FIXED** `e55efc6` (owner chose fix-now): a `NOT EXISTS` self-join latest-accepted-wins guard on the UPDATE (writes only if no other accepted quote is strictly newer by `(accepted_at, id)`). **Verified 5/5 against real Postgres** (local docker test DB) incl. the older-no-op / newer-wins race case.
- **Medium** — Backfill write TOCTOU: the dealer guard was only in the candidate SELECT; the write updated by `campaigns.id` alone. **→ FIXED** `249560c` (write UPDATE now carries `campaigns.dealerId = item.dealerId`).
- **Non-issues confirmed:** the added `dealerId` select is null-safe (both cols `notNull()`); `setQuoteDealer` rejects non-draft; billing overlay still composes; `DISTINCT ON` latest-pick correct.
