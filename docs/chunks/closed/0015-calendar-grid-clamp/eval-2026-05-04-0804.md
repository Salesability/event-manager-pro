### Low

**Missing regression coverage for events that overlap only the leading or trailing strip**

- File: `src/app/(app)/calendar/calendar-grid.test.ts:29`
- The new tests cover ranges from outside into the main month and from the main month out past the grid, but not the exact failure shape where the visible slice exists only in the 42-cell grid's *other-month* cells. Current coverage includes:

  ```ts
  expect(clampToGrid('2026-04-20', '2026-05-05', grid)).toEqual({ si: 0, ei: 9 });
  expect(clampToGrid('2026-05-28', '2026-06-10', grid)).toEqual({ si: 32, ei: 41 });
  ```

  These do not prove `2026-04-20 → 2026-04-28` clamps to `{ si: 0, ei: 2 }` or `2026-06-03 → 2026-06-20` clamps to `{ si: 38, ei: 41 }`, even though those leading/trailing strip-only spans are part of the stated bug.
- Severity rationale: a test gap, not an observed runtime defect; the implementation appears to handle these cases today. It matters because those cases are closest to the original cross-month rendering failure.
- Suggested fix: add two focused tests — prior-month-only overlap with the leading strip, and next-month-only overlap with the trailing strip.

### Things explicitly ruled out (false-positive reduction)

- **Record mutation / cross-month state bleed** — `slotAssignment` copies each campaign before adding `_si`/`_ei` via `{ ...c, _rowSlotAssigned: {}, _si: clamped.si, _ei: clamped.ei }`; original `campaigns` references aren't mutated across month changes.
- **Slot packer vs ribbon drift** — slot assignment reads `ev._si`/`_ei` at lines 230, 243-250, and 261; `drawRibbons` reads the same fields at 321-322. Same source of truth.
- **DST / local TZ fragility in the clamp** — `clampToGrid` compares stored `YYYY-MM-DD` strings lexicographically; doesn't construct `Date` objects.
- **Inclusive-range overlap test** — `si <= oei && ei >= osi` is correct for inclusive start/end cell indices, including single-cell and zero-day events.
- **`GRID_LAST_INDEX = 41`** — `CalendarView` always builds 42 cells before deriving `lastDate`; the exported helper would be less general if reused, but this chunk's invariant is explicitly 6×7.
- **Fully before / fully after grid** — covered by tests at lines 21-27 and the early returns in `clampToGrid`.
- **Exact-boundary zero-day events** — covered for cells 0 and 41 by tests at lines 48-50.
