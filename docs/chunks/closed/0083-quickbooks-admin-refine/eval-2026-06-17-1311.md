## Codex Adversarial Review

**High:** No findings.
**Medium:** No findings.

**Low:**
- `dealer-sync.ts` `decodeSyncSummary` / `item-sync.ts` `decodeItemSyncSummary` (reused by `qb-sync-summary.ts` `decodeQbSyncSummary`) — oversized all-digit `?qbsync=` segments were accepted instead of rejected: `Number.parseInt('9'.repeat(309), 10)` overflows the double to `Infinity`, which passes the `/^\d+$/` gate and yields a non-null summary. Impact limited to flash-notice display (admin-only page), not mutation. **→ FIXED in-cycle at `80e785c`:** both decoders now reject any parsed segment that isn't `Number.isSafeInteger`; overflow test cases added to all three summary test files.

**Explicitly NOT an issue (Codex false-positive reduction — confirmed sound):**
- **Partial-report independence** — dealer + item passes are in separate try/catch; dealer writes are outside the item `db.transaction`, so an item failure rolls back item writes without touching already-committed dealer writes. No double-count or cross-corruption.
- **Token/auth-gate placement** — `assertCan('admin:access')` runs before `getValidAccessToken()`, and the token fetch is outside the per-pass catches (intentional: a not-connected/refresh failure propagates → keeps the gate-matrix admin row green).
- **URL error-message encoding** — the redirect is built via `URLSearchParams`, so `?qbderror=`/`?qbierror=` messages can't break out into extra query params.
- **Summary segment count + common tampering** — exactly 7 dot segments required; negative / non-numeric / `NaN` / `Infinity` / scientific-notation rejected.
- **RSC/client boundary** — `quickbooks-tabs.tsx` takes `React.ReactNode` panels; no server action threaded through the client tab component.
- **Removed-action references** — no live `src/**` references to the deleted `syncDealersFromQuickbooks` / `pullItemsFromQuickbooks`; the matrix row now invokes `syncQuickbooks`, drift detection coverage did not regress.
