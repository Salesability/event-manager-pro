## Codex Adversarial Review

**Critical:** None.
**High:** None.
**Medium:** None.

**Low**

1. **Search input has no stable accessible label** — `src/features/people/people-admin.tsx:154`
   The placeholder was doing the labeling work. Suggested fix: add `aria-label="Search people"`.
   **Status: FIXED in this eval pass.** `aria-label="Search people"` added.

**Not an issue (false positives ruled out)**

- `FilterFn` signatures: TanStack's `filterValue` type is `any`; the runtime predicates fail-open for unexpected values (`Array.isArray` check on roles; sentinel string match on `dealerLinks`).
- Pill toggle race / stale state: all toggles use functional `setColumnFilters(prev => ...)` so they compose against the latest state.
- Pagination reset on filter change: TanStack auto-resets page index when filtered rows change (`autoResetPageIndex` is on by default since `manualPagination` isn't set).
- Search clear-X behavior: `type="search"` browser clear fires onChange with `''` — matches the Clear filters path naturally.
- `aria-pressed` correctness: all three pills expose `aria-pressed={…Active}` matching toggle-button semantics.
- Empty-state `Clear filters` reachability: rendered as a real `<button type="button">` inside the empty cell, focusable/keyboard-reachable.
- Dealer-name substring search: query lowercased + trimmed; nullable email guarded; `dealerLinks[]` is always non-null per the type.
- Role OR semantics: `some(...)` inside the column filter combined with TanStack's per-column-AND gives Coach + Admin = "either role" while composing AND with Customer-side.
- `columns` useMemo stale closure: in current Next 16.2.4 `useRouter()` returns a stable singleton; `archive()` captures only stable refs. The eslint-disable comment is inaccurately worded but no defect.
