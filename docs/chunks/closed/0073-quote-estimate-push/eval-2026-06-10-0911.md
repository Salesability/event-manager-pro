## Codex Adversarial Review

(Verbatim severity; disposition added by the orchestrator.)

### High
None.

### Medium

1. **Update path may not replace an existing Estimate's line set** (`quote-push.ts` update branch + `client.ts` `updateEstimate` sparse). `fetchEstimateById` is used only for the `SyncToken`; the existing QBO `Line[].Id`s are ignored and `updateEstimate` always sends `sparse: true`. For an Estimate the child line collection is the commercial payload — a sparse update *may* leave stale/append lines rather than replacing them on re-push.
   - **Disposition: PARK (0073 follow-up a).** Hypothesis — needs a live sandbox round-trip to confirm QBO's sparse-update line semantics (we kept the eval read-only, no QBO write). Only affects the **re-push/update** path; the create (first push) path is correct. Fix direction: verify in sandbox; if sparse doesn't replace the line collection, switch the update to a **full** update (or send line ids / deletion semantics) so the Estimate exactly mirrors the quote. **Un-park before relying on re-push / before prod use.**

2. **Cents mismatch can produce `Amount != Qty × UnitPrice`** (`mapQuoteToEstimate`). The quote-write validator (`assertNonNegMoney`) doesn't cap decimals, so a >2-decimal price stores `unit_price` rounded (numeric(10,2)) while `line_total` was computed pre-round — so `Amount = lineTotal` could differ from `Qty × UnitPrice`, which QBO validates.
   - **Disposition: FIXED IN-CYCLE at `308d5a8`.** `mapQuoteToEstimate` now derives `Amount = round(unit × qty)` (self-consistent for QBO) instead of `Number(lineTotal)`. Identical to `lineTotal` in the normal ≤2-decimal case; diverges by at most a cent only in the rare >2-decimal entry, and a self-consistent line keeps QBO from rejecting/recomputing.

### Low
None.

- SyncToken read-before-write + guarded `IS NULL` backfill match dealer-push's accepted orphan-on-lost-race trade-off.
- Additive `Quote.quickbooksEstimateId` didn't break a production literal-construction (the one test fixture was fixed; `queries.test.ts` passes).
