## Codex Adversarial Review

### Pass 1 ‚Äî 3 Medium + 2 Low (self-resolved)

**Medium**

1. `quotes-filters.tsx:45` ‚Äî `pushParams()` closes over the `params` snapshot from the render that scheduled the debounced timeout; if a user types, then clicks a pill within 250 ms, the later timeout can replace the URL from the stale params and remove `status`. Real stale-state bug in combined filter semantics.
2. `page.tsx:7` ‚Äî `searchParams` typed only as `string`, but Next page `searchParams` values can be `string | string[] | undefined`. A URL like `/quotes?q=a&q=b` would pass an array into `filterQuotes`, where `q.trim()` throws.
3. `app-nav.tsx:20` ‚Äî `Quotes` exposed to every staff-app role because the nav only understands `requiresAdmin`. `staff` and `viewer` would pass the app shell but fail `assertCan('quote:edit')`, getting a visible tab that redirects away.

**Low**

1. `quotes-filters.tsx:33` ‚Äî URL `status` cast to `Pill` without validation; `?status=foo` rendered no active pill (including `All`) and retained the invalid value across search changes.
2. `page.tsx:61` ‚Äî empty action-column header + generic `View` text; screen-reader users got a weaker action/row relationship.

### Fixes applied (Codex write-capable rescue)

- `quotes-filters.tsx:51` ‚Äî `pushParams` reads `new URLSearchParams(window.location.search)` instead of the closure'd `params`. Fixes Medium 1.
- `page.tsx:7,17-18,28,79-81` ‚Äî `searchParams` retyped `string | string[]`; new `pickFirst` helper applied before `filterQuotes`. Fixes Medium 2.
- `app-nav.tsx:9,17-23,40-44` ‚Äî `Tab` extended with `capability?: Capability`; `Quotes` set to `capability: 'quote:edit'`; filter logic uses `ctx?.can` from `CapabilityContext` (post-orchestrator-correction ‚Äî see below). Fixes Medium 3.
- `quotes-filters.tsx:23-25,37-38` ‚Äî `isPill` guard added; invalid status falls back to `''`. Fixes Low 1.
- `page.tsx:63` + `row-actions.tsx:13` ‚Äî `sr-only` `Actions` header + per-row `aria-label="View ${quote.dealerName}'s quote"`. Fixes Low 2.

### Orchestrator correction (rules-of-hooks)

Codex's first attempt at the Medium 3 fix called `useCan(t.capability)` inside `.filter()` ‚Äî a rules-of-hooks violation that would have failed `react-hooks/rules-of-hooks` lint. The orchestrator replaced it with `useContext(CapabilityContext)` at the top of `AppNav` + a stable `can = ctx?.can ?? (() => false)` predicate that's safe to call inside the filter callback. Functionally equivalent (`useCan` is `useContext(CapabilityContext)` + `ctx.can()` under the hood; the orchestrator's version just lifts the hook out of the callback).
