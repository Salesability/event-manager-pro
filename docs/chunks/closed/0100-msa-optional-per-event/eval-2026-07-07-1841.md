## Codex Adversarial Review

**High — FIXED IN-CYCLE (`2686fbd`)**

`src/features/quotes/accept-gate.ts` — Waiver skip was not dealer-scoped, so a stale cross-dealer `campaignId` became an MSA accept-gate bypass. Break scenario: a draft quote for Dealer A's *waived* campaign, then `setQuoteDealer` swaps it to Dealer B (no MSA) — the quote keeps A's `campaignId`, so `isAcceptMsaSatisfied(B, campaignId_A)` read A's waiver and returned `true`, bypassing B's active-MSA requirement. 0100 turned the parked-0094-c stale-link bug into a hard-gate bypass.
**Fix:** dealer-scope the waiver lookup — `where(and(eq(campaigns.id, campaignId), eq(campaigns.dealerId, dealerId)))` in `isAcceptMsaSatisfied`, and the matching dealer-scoped join in `loadQuoteEventMsaWaived` (UI mirror). A cross-dealer link now matches 0 rows → the waiver isn't honoured → falls through to the dealer's own MSA check. Mirrors 0094's cross-dealer write guard. New integration test case `does NOT honour a cross-dealer waiver` proves it.

**Medium — PARKED (0100-a)**

`src/features/quotes/actions.ts:~1289` — The MSA/waiver gate is checked outside the atomic `sent → accepted` UPDATE, so a concurrent `setMsaWaived(false)` can race acceptance: A reads waived=true → passes the gate; B un-waives; `markQuoteAccepted` (which guards only status + expiry) still commits. **Pre-existing property of the 0082 accept gate** — the MSA check has always been non-atomic w.r.t. the accept (a concurrent MSA termination raced the original gate the same way); 0100 only adds the waiver as another thing that can change in the sub-millisecond window, and it needs two admins acting on the same event concurrently. Fixing it properly means folding the MSA/waiver predicate into `markQuoteAccepted`'s atomic UPDATE, which touches the 0082 gate broadly → out of 0100's scope.

**Low:** none.

**Not an issue (affirmatively verified by Codex):**
- `setMsaWaived` authz — `campaign:edit` resolves admin-only; no non-admin path.
- Null `campaignId` → correctly falls back to the normal active-MSA gate.
- Expired-but-still-`active` MSA rows → correctly blocked by `expires_at >= now()`.
- Exposed-flag change leaves the quote dimension intact (a non-accepted quote stays exposed).
- Guarded `setMsaWaived` UPDATE `.returning()` → no-op on a bad id.
