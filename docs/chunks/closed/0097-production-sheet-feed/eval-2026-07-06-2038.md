## Codex Adversarial Review

**Medium**

- `route.ts:24` / `production-feed.ts:39`: `todayIso()` uses the host process timezone, then `selectFeedCampaigns` filters `endDate >= todayIso`. If prod runs UTC while the business day is Atlantic, a campaign ending "today" locally can drop from the feed after 00:00 UTC but before local midnight. Confirmed code issue; deployment TZ unverified.
  - **Disposition: parked as 0097-a.** This is the **app-wide** "today" pattern — `src/app/(app)/production/filter.ts` `todayIso`, the CSV export route, and the calendar all derive today from host TZ, so `/production` and the feed agree. Pinning only the feed to Atlantic would make the feed *disagree* with the list; a correct fix is an app-wide TZ decision, out of this chunk's scope. Impact is a few evening hours of early-drop on an ~hourly vendor feed.

**Low (accepted, no action)**

- `route.ts:30`: `tokenMatches` returns on length mismatch before `timingSafeEqual`, so it isn't strictly constant-time for wrong-length tokens (leaks expected length). Matches the existing `boldsign/webhook-verify` idiom; doesn't authorize, doesn't read the DB, token is fixed-length hex → accepted.
- `middleware.ts:18`: `/api/production-feed/*` is also public via the `startsWith(p + '/')` guard. The slash guard prevents `/api/production-feed-admin` confusion (verified by the near-miss smoke → 307); it's the same intended behavior as every `PUBLIC_PATHS` entry (e.g. `/share/coach/*`). No nested feed routes exist → accepted; note for any future nested route under the feed.

**Confirmed NOT issues (Codex-verified)**

- Empty/unset `PRODUCTION_FEED_TOKEN` rejected before reading `?token=` (route.ts:41).
- Duplicate `?token=` params not a bypass (`searchParams.get` takes one value, still must match).
- Redaction explicit — the mapper emits only dates/dealer/location/format/coach/volumes; never notes/contact/phone/email/audience source.
- `csvCell` applied to header AND every data cell → formula-injection safe in the consuming Sheet.
- Route handler is the right choice for the external `IMPORTDATA` caller; `dynamic='force-dynamic'` + `Cache-Control: no-store` → no stale/cross-user caching.
- Admin-panel token exposure limited to the `admin:access`-gated `/production` (intended; token also lives in the Sheet formula + logs, as documented).
