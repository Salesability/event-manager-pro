## Codex Adversarial Review

3 findings — 0 High / 2 Medium / 1 Low.

### Medium

**1. Production status filtering can drift from the rendered/exported status date** — `src/app/(app)/production/production-columns.tsx`

> The column is built with a request-time `todayIso` and uses it for display (`const past = c.endDate < todayIso`), but the filter ignored that value and recomputed a browser-local date (`const today = todayIsoSafe()`). CSV export uses the server-side date again. Around midnight or when the deployment server timezone differs from the browser, the UI filter can include a campaign that the badge/export classify differently.

**Fixed in-cycle at [`9db4298`](#).** Moved `filterTimeStatus` from module level into a closure inside `buildProductionColumns(actions, todayIso)` so it captures the same `todayIso` the cell render uses. Retired the `todayIsoSafe()` local helper.

**2. Report dealer identity links point at a route some report users cannot open, and archived dealers can 404** — `src/features/reports/reports-columns.tsx`

> `/reports` is gated by `await assertCan('reports:view')` (admits coaches per `capabilities.ts:79`), but `/dealerships/[id]` requires `await assertCan('admin:access')`. The new drill-through (dealer aggregate tab + full production report tab) is unconditional. A coach clicking the dotted-underline routes to a 403. Separately, report queries include campaign-linked dealers without filtering archived rows, while `loadDealer()` filters `isNull(dealers.archivedAt)`, so archived historical report rows render a dotted link to a guaranteed not-found page even for admins.

**Fixed in-cycle at [`9db4298`](#).** Retired the `/reports` drill-through entirely (dealer aggregate tab + full production report). Group/dealer labels render as plain text. `docs/wiki/layout.md` updated to capture the three reasons the exception holds: (a) no `/coaches/[id]` or `/reports/[month]` page, (b) coach-vs-admin capability mismatch on `/dealerships/[id]`, (c) archived-dealer 404. Revisit when either a coach-safe dealer surface or an archived-loader exists.

### Low

**1. TokenPill truncation can split surrogate pairs if a future caller passes non-ASCII opaque values** — `src/components/ui/token-pill.tsx`

> The truncation uses UTF-16 code units (`value.slice(0, maxChars)`). A future value containing an astral character (4-byte emoji, e.g. 🎉 with `.length === 2`) on the boundary would be sliced mid-pair and render a replacement character. This is fine for today's unused ASCII-token intent.

**Parked as 0050 follow-up (a).** Primitive has no consumer today; hardening lands when the first surface adopts it. Fix shape: use `Intl.Segmenter` or iterate code points (`Array.from(value).slice(0, maxChars).join('')`) instead of UTF-16 `.slice()`.
