## Codex Adversarial Review

0 High, 0 Low. Two Mediums:

1. **Rate-snapshot TOCTOU** (`actions.ts` createQuote/applyPickerSave/setQuoteTax + `tax-rates/queries.ts:36`). `dealerTaxRatePct` reads the dealer's province→rate on the **app pool** before/outside the quote tx that locks the dealer (`FOR UPDATE`). A concurrent `updateDealer` changing the province between the rate read and the lock can snapshot a stale `tax_pct` (e.g. ON 13% stored on a now-QC dealer). Low-probability and **self-healing** (next picker save / send re-derives), but a genuine money-path gap. **→ Parked as `0065 follow-up (a)`.**
2. **Migration locks existing drafts into manual-override mode** (`0028 … UPDATE quotes SET tax_override = tax`). Makes every pre-existing quote — including drafts at $0 — a manual-tax quote, so an old draft won't auto-compute province tax until the coach clears the field. Codex notes this is **"consistent with the stated 'existing quotes unchanged' intent."** **→ Accepted** as the documented behavior (intent open-question lean: preserve existing tax). A future tweak could null-out drafts only.

Codex confirmed as **not issues**: override-of-0 handling (`'0'` truthy → kept), pricing precedence/rounding (override wins incl. 0, else `roundCents(subtotal×ratePct/100)`), `setQuoteDealer` preserving an existing override while re-deriving for the new province, and `updateTaxRate` gating (`lookup:edit` + enum/regex/range validation + Drizzle parameterization).
