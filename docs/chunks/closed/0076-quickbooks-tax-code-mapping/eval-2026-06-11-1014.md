## Codex Adversarial Review

### Pass 1 (full chunk diff) — verbatim

**High — `assignProvinceTaxCode` could map a code with no resolvable rate.** `resolveCodeRatePct` → `if (ratePct != null) set.rate = …` still wrote `{ quickbooksTaxCodeId }`; an active code with no `SalesTaxRateList` (or a member `TaxRateRef` missing from `fetchTaxRates`) → province becomes "managed" with a stale rate; the push pre-flight could pass on the stale rate while QBO computes differently.

**Medium — `refreshTaxRates` TOCTOU.** Refresh wrote by province only; a concurrent re-map (ON: code A → B) between planning and commit could write A's rate onto B's mapping.

**Low (parked `0076-a`) — `loadTaxMappingAdmin` collapses any QBO/token/fetch error to `{ connected: false }`**, so the UI tells the admin to "Connect QuickBooks" even when connected-but-QBO-errored, hiding drift/broken-link diagnostics.

**Not bugs (confirmed):** keep-13-rows + seeded-fallback (matches `decision.md`); `planRateRefresh` never remaps; the `w.province as CaProvinceCode` cast is safe (DB enum source); QBO code names are React-escaped in `<option>` (no XSS); the client component imports only types from the server `mapping.ts`; `applyTaxCodeSync`/encode-decode retained-but-unwired is tested, not a dangling path; `pullTaxCodesFromQuickbooks` fully removed from production surfaces.

### In-cycle fix — `c4a07b5`
- **High:** `assignProvinceTaxCode` now returns an error when `ratePct == null` (writes nothing), and always writes `rate` on success. `buildTaxCodeOptions` filters `o.ratePct != null` (keeps 0%, drops null) so unassignable codes aren't offered.
- **Medium:** `RateRefreshResult.writes` carries `quickbooksTaxCodeId`; the executor's `.where()` is `and(eq(province), eq(quickbooksTaxCodeId, expected))` — a compare-and-set that no-ops if the mapping changed.

### Pass 2 (re-review of the fixes) — verbatim
- **Prior High — FULLY RESOLVED.** Rejects unresolved rates before writing; writes both fields together; unmap path intact; the filter drops only `null`, keeps `0`.
- **Prior Medium — FULLY RESOLVED.** `planRateRefresh` carries a non-null code id; the executor CAS matches both `province` and `quickbooksTaxCodeId`; `toFixed(3)` matches the `numeric(6,3)` column.
- **New Low (parked `0076-b`) — `refreshTaxRates` reports `updated: writes.length` (planned), not rows actually updated** after the CAS; if a concurrent remap voids a write the reported count overcounts. (DB state stays correct — only the toast count is optimistic.)
