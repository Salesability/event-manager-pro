## Codex Adversarial Review

**Critical:** None.

**High:**

1. **Replay/flood is still open from 0011's original Highs.** A booked campaign can be emailed repeatedly; coach share-link sends have no rate-limit/idempotency/audit guard before the external side effect. **Status: out of Phase 7 scope.** Phase 7 explicitly addressed three Codex findings from `closed/0011-email-send/eval-2026-05-01-1609.md` — Host-header allowlist (High #2), status check (Medium), dev-redirect inversion (Medium). The replay/rate-limit High (#1 in that eval) was NOT in Phase 7's checklist; it remains parked under the bigger "auth/abuse hardening" theme. Worth a dedicated chunk before public exposure (e.g., booking intake at 0016 will need the same rate-limit infra).

**Medium:**

2. **`APP_ENV` brittle.** `APP_ENV=PRODUCTION` (uppercase) or ` production ` (whitespace) is treated as non-production, with no `EMAIL_DEV_TO` → refuse (fail-closed, OK), or with `EMAIL_DEV_TO` → silent dev-redirect on a real-prod deploy (bad). **Status: fixed in-eval.** `decideRedirect()` now uses `.trim().toLowerCase()` before comparing. Added a vitest case for `APP_ENV=' Production '` to lock in the normalisation.

3. **Auth still has the same Host-header callback surface.** `src/features/auth/actions.ts:13`'s `siteUrl()` falls back to `headers()` for `signInWithMagicLink` / `signInWithGoogle` callbacks. **Status: deferred.** Out of Phase 7 scope (this phase covers email-to-recipient, not OAuth/magic-link callbacks). Supabase redirect allowlisting blunts most of the impact, but tightening it in the same way (require `SITE_URL` env, no header fallback) is a small follow-up. Worth a separate chunk or fold into a v2 of 0019.

**Low:**

4. **SITE_URL test relied on visual review for URL shape.** The original test mocked `coachShareLink` to ignore args, so it didn't actually prove `${origin}/share/coach/${id}` got constructed correctly. **Status: fixed in-eval.** Captured `coachShareLink`'s args via a `vi.fn().mockReturnValue(...)` and asserted `shareUrl === 'https://app.example.test/share/coach/7'` literally, including the trailing-slash-strip variant.

5. **0016-book-your-event-intake plan referenced the retired `EMAIL_FORCE_DEV_REDIRECT` flag.** **Status: fixed in-eval.** Updated `docs/chunks/0016-book-your-event-intake/plan.md:90` to point at the new `EMAIL_DEV_TO`-based convention and note the 0019 Phase 7 retirement.

**Explicitly cleared (not issues):**
- APP_ENV unset + EMAIL_DEV_TO unset → refuse-before-Resend confirmed at `lib/email/send.ts:53`, covered by test at `send.test.ts:92`.
- SITE_URL as "operator-set allowlist" is acceptable for closing the Host-header issue. A static origin allowlist in code would be belt-and-suspenders against operator misconfig but adds maintenance burden — Codex agreed not to add one. Worth considering URL-shape validation (require http/https, no path) as a small add later.
- Status-gate race (cancel between `loadCampaign` and `sendEmail`): real but not material to the parked replay finding. Fixing properly wants an outbox/idempotency design, not just another read.
- `.env.local` doesn't set `EMAIL_FORCE_DEV_REDIRECT`; `.env.example` accurately documents the new behaviour.
