## Codex Adversarial Review

**Critical:** None.

**High:** None.

**Medium:**

1. **Non-admin update/archive has a row-ownership TOCTOU window.** `src/features/schedule/actions.ts` (update + archive paths). A coach's pre-read verifies the row is currently their own, but the later UPDATE/archive `WHERE` only checked `id` + `archivedAt`; if an admin concurrently transferred or changed the row between those statements, the coach write could still apply. Real race but requires concurrent privileged mutation. **Status: fixed in-eval.** Added `kind = 'coach_unavailable' AND coach_id = myCoachId` to the non-admin `WHERE` in both `updateAvailabilityBlock` and `archiveAvailabilityBlock`. Admin path retains the existing two-predicate `WHERE` (no regression). `myCoachId` resolved at the action level via `loadCurrentMembership()` (request-cached, so no extra DB hit beyond what the helper would have done).

**Low:**

2. **Membership revocation can be stale within one request.** `src/lib/auth/load-team-membership.ts:26` (React `cache()` request-scoped). If a coach role is revoked concurrently after the first membership read in the same request, the ownership helper reuses stale membership; new requests refresh. **Status: deferred.** Closing this would require giving up the request-cache (per-action membership lookups, ~+1 DB roundtrip per call) for a vanishingly narrow race. Not worth the cost given staff-churn frequency.

**Explicitly cleared (not issues):**
- `coach_id` semantics — `availabilityBlocks.coachId` references `contacts.id`, same identity space as `coachContactId`.
- Null/unknown `kind` — schema enum is NOT NULL; helper denies anything ≠ `coach_unavailable`.
- `existing` null — both update and archive return `"Availability block not found."` before calling the helper.
- Zero facets — helper trivially allows (admin returns null; non-admin's for-loop is a no-op). Footgun if more callers appear; today's three callers always pass ≥ 1 facet.
- Admin existing-load bypass — no regression vs. pre-Phase-3.
- Create validate/ownership race — covered by the Low note above.

Codex verdict: "Ship-with-followups. Phase 3 closes the direct cross-coach mutation bug."
