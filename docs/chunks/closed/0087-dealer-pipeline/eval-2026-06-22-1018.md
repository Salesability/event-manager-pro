## Codex Adversarial Review

0 High / 3 Medium / 1 Low. **All four fixed in-cycle at `441a415`.**

- **[Medium → FIXED] setDealerPipeline active-lock TOCTOU.** The select-then-update let a concurrent `convertProspectToActive` slip a pipeline edit onto a now-active dealer. Fix: guarded the UPDATE on `status='prospect'` + `returning()`; zero rows ⇒ "Pipeline is locked once a dealer is active." (mirrors the `updateDealer` guarded-UPDATE pattern).
- **[Medium → FIXED] ownerId not coach-validated.** Decision D2 requires action-layer coaches-only enforcement; only UUID/FK validation was present, so a forged admin request could set any user as owner. Fix: new `ownerIsCoach(userId)` check in `setDealerPipeline` → "Owner must be a coach."
- **[Medium → FIXED] date regex accepted impossible dates** (`2026-02-31` silently rolled to Mar 3; `2026-99-99` would make Postgres throw). Fix: `isRealIsoDate` round-trip validation in `pipeline-schema.ts` (rejects, returns "Enter a valid date.").
- **[Low → FIXED] logDealerActivity concurrent-archive insert.** A concurrent archive could leave an activity orphaned on an archived dealer. Fix: reordered to guarded dealer UPDATE-first + `returning()`; zero rows ⇒ skip the insert, return not-found.

Codex explicitly flagged as **NOT issues** (verified): `next_action_at` string-mode consistency across schema/query/zod/UI; D4 (no notes append); the migration backfill scope (`status='prospect' AND pipeline_stage IS NULL`); lexical date comparison in the queue; the `9999-12-31` no-due sort sentinel.
