## Codex Adversarial Review

**High** — None. **Medium** — None. **Low** — None.

Confirmed NOT an issue (verbatim points):
- **Auth/data leak** — `page.tsx` calls `assertCan('admin:access')` before `loadServiceItemsForAdmin()`. Gate in place.
- **Read-only surface** — `service-items-list.tsx` is a server component, no forms/buttons/actions.
- **XSS/injection** — DB values (`code`/`label`/`unitPrice`) render as React text (auto-escaped); no `dangerouslySetInnerHTML`.
- **Null handling** — `unitPrice === null` → "variable"; `archivedAt === null` → "Active". Both handled. Empty list → "No service items in the catalog." (`colSpan=5`), no crash.
- **Loader shape/perf** — single select, intentionally unfiltered (returns archived), ordered by non-null `code`. No N+1, no null-order throw.
- **QBO-disconnected render** — catalog loads before the `if (conn)` block; independent of tokens/QBO fetches → safe when disconnected.
- **Test-fix** — `toBeGreaterThanOrEqual(1)` doesn't mask a regression: the seeded row's `archivedAt` is still asserted non-null directly.
