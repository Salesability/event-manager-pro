## Codex Adversarial Review

### Pass 1 (full chunk diff)

**High** — `dealer-pipeline-panel.tsx`: `nextForm` was seeded from `dealer.nextAction` only at mount. After `submitDone` advanced/cleared the commitment via `logDealerActivity` + `router.refresh()`, the idle/edit form still held the stale value and could re-submit it through `submitNextAction` → `setDealerPipeline` (writes whenever `formData.has('nextAction')`), silently restoring/clobbering the refreshed commitment.
→ **FIXED in-cycle (`c9b0b15`):** added a `useEffect` keyed on `[dealer.nextAction, dealer.nextActionAt, nextFormReset]` that re-syncs `nextForm` to the refreshed `dealer` prop. The submit handlers flip `heroMode` to `'view'` before `router.refresh()`, so the reset always lands on a closed editor (no in-progress-edit wipe).

**Medium** — *false positive (confirmed)*: no omit-when-absent field-set bug. `submitDone` sends `nextAction`/`nextActionAt` + omits `occurredAt`; `submitNextAction` sends only commitment fields; `submitDetails` omits commitment fields; `submitLog` omits commitment fields — all aligned with the server `formData.has(...)` patch semantics. No action.

**Low** — date inputs (`nextActionAt`, `doneForm.nextActionAt`, `logForm.occurredAt`) render no `FieldError`/`aria-invalid`, so a `dateOrEmpty`-rejected value would block submit silently.
→ **Parked (0090-a).** Pre-existing (the 0087 panel was identical) + not reachable through a native `<input type="date">` (the browser enforces valid YYYY-MM-DD); the schema round-trip rejection is a forged-input backstop, not a UI path.

### Pass 2 (re-verify of the fix)

- Stale-form clobber **resolved** — no finding.
- Mid-edit wipe — **no confirmed bug**; the submit paths close the editor before refresh, and `dealer.nextAction` cannot change while a form is open except via this panel's own submits (no polling/subscription/websocket in the app).
- `nextForm.reset` is a **stable ref** (RHF stores form methods in a ref) — the effect won't loop.
