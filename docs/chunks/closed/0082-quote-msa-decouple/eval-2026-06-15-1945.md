## Codex Adversarial Review

3 High, 0 Medium, 0 Low. Verbatim findings + dispositions:

**High #1 — MSA send still `msa:edit` (coach-admitting) but the only UI is admin-only** (`src/features/msa/actions.ts:80,149`). A coach could bypass the admin-only dealer page and invoke `createMsaDraft`/`sendMsaEnvelope` directly. → **FIXED in-cycle (`72c2231`):** both actions retightened to `admin:access`, matching D2's recorded admin-only intent. `msa:edit` is now defined-but-unused (kept for forward-compat; its `can()` tests still pass).

**High #3 — accept gate checks `status='active'` but ignores `expires_at`** (`src/features/quotes/actions.ts`). The expiry sweep that flips `active → expired` isn't built, so an expired MSA can still read `active` and let a quote be accepted past the 12-month term. → **FIXED in-cycle (`72c2231`):** the gate query now also requires `expires_at >= now()` (Postgres-time, like the quote expiry guard); a null/past expiry blocks. (Codex's secondary "outside the atomic transition" TOCTOU note — concurrent MSA *termination* between the pre-read and the accept UPDATE — is a negligible-probability edge left as the parked nuance below; the pre-read expiry predicate closes the practical hole.)

**High #2 — `sendMsaEnvelope` not idempotent under concurrent sends** (`src/features/msa/actions.ts`). The BoldSign `sendSignatureRequest` call happens before the guarded `providerDocumentId` persist; two concurrent callers can post two envelopes, only one id saved, the orphan's signed-webhook 404s. → **PARKED (pre-existing).** This send-then-guard ordering predates 0082 (the decouple didn't touch it); after Fix #1 the action is admin-only and fires from a confirm dialog that disables while pending, so a concurrent double-send needs two simultaneous admin sessions racing — low probability. See **Parked** below.

**Not a bug (false positives ruled out by Codex)** — all validate the chunk:
- `quotes.msa_id` fully removed; migration `0040` order correct (drop FK → index → column).
- `sendMsaEnvelope` payload is genuinely MSA-only (no quote PDF / initials anchors).
- `markMsaSigned` / the webhook no longer touch quotes.
- Reusing the pre-loaded `dealerId` for the prospect promotion is not stale (dealer-swap only on `draft`; accept only on `sent`).
- The `sendQuote` send optimistic-lock survived the re-send-guard removal.
