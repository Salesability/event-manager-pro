## Codex Adversarial Review

### High — Envelope can be sent before the quote is linked, so signing may not auto-accept
`src/features/msa/actions.ts:257/340/383`. The `quotes.msaId` link UPDATE happens *after* the BoldSign envelope is posted, and its result is ignored; if the quote changes status (accepted/declined) or is linked by another path between the preload and the final guarded update, BoldSign gets a valid envelope but `acceptBundledQuote` later finds no row → MSA activates, bundled quote never auto-accepts. Codex's fix: claim/link the quote with a guarded `UPDATE … RETURNING` *before* external side effects; abort if no row returned.

**Disposition: PARKED (pre-existing, out of 0061 scope).** This link-after-send-with-ignored-result ordering predates 0061 (shipped in 0055/0041 with the draft-only guard); 0061 only widened the status guard from `draft` to `draft|sent` (which actually removes one race variant — a concurrent draft→sent no longer breaks the link). It is the same family as the already-parked **0041 follow-up (a)** ("concurrent-send race on `sendMsaEnvelope` — claim the send before external side effects"). Tracked there; not a regression introduced by this chunk.

### Medium — Bundle button ignored unsaved quote edits (stale pricing in a signed doc)
`src/features/quotes/quote-composer.tsx`, `src/features/msa/msa-send-button.tsx`. The "Send Quote" button is disabled on `isDirty` (it emits the persisted snapshot), but the new "Send for signature" button had no such guard — a coach could edit pricing, skip saving, and send the MSA+Quote envelope, which `sendMsaEnvelope` renders from the *persisted* `quote.lineItems`, baking stale pricing into the legally-signed artifact.

**Disposition: FIXED IN-CYCLE (`099d6a4`).** Added `disabled={isDirty || pending || sendPending}` + a "Save changes before sending for signature…" tooltip to `MsaSendForSignatureButton`, mirroring the Send-Quote guard. The existing "You have unsaved changes — save before sending" banner (gated on `canSend && isDirty`) already covers the bundle case.

### Explicitly NOT an issue (Codex-confirmed)
- The four draft→draft|sent guards are consistent (`sendMsaEnvelope` status check + link UPDATE; `acceptBundledQuote` SELECT; `markQuoteAcceptedViaEnvelope` UPDATE).
- No other production caller of `markQuoteAcceptedViaEnvelope`; the omitted expiry guard is intentional + documented.
- Auth model matches the move (`quote:edit` page gate; `msa:edit` action gate; both admin||coach).
- Dealer-page removal is clean — no dangling imports of the deleted `MsaCreateTrigger` / `msa-panel` / `firstDraftQuoteForDealer`.
