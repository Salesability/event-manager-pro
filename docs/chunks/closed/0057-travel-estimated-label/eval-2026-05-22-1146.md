### High — Migration would silently never apply on the shared DB (FIXED in-cycle)

- drizzle migrate gate (`node_modules/drizzle-orm/pg-core/dialect.js:62`): applies a migration only if `lastDbMigration.created_at < migration.folderMillis`, where `lastDbMigration` (the newest `__drizzle_migrations` row by `created_at`) is read **once** before the apply loop.
- Journal had 0021 `when=1779984000000`, 0022 `when=1779464350761` (earlier — `drizzle-kit --custom` stamped the real wall-clock, but 0019–0021 carry artificially future-dated timestamps).
- On the shared DB (last applied = 0021, stored `created_at=1779984000000`), the gate `1779984000000 < 1779464350761` is **false** → 0022 skipped forever.

**Fix (`3404fae`):** bumped 0022 `when` to `1780070400000` (> 0021), restoring strict monotonicity across all 23 entries. Verified the journal is valid JSON and strictly increasing.
