## Codex Adversarial Review

**Critical / High:** none.

**Medium (2 found, both fixed in-eval):**

1. **Gate-before-loader invariant untested (FIXED).** Original "asserts the staff gate" test only checked `requireRole(...)` was called — a future refactor that loaded data before awaiting the gate would still pass. Resolution: assert `mocks.requireRole.mock.invocationCallOrder[0] < mocks.loadCampaignsByDealer.mock.invocationCallOrder[0]` so reordering breaks the test.
2. **Rejection path uncovered (FIXED).** All tests mocked `requireRole` as success; none made it throw like real `redirect()` does. Resolution: added a dedicated test that mocks `requireRole.mockRejectedValue(...)` and asserts (a) `GET()` throws, (b) NONE of the four loaders fired. Proves unauthorized requests short-circuit before any I/O.

**Low (1, deferred — Phase 1 carry-forward):**

- Aggregate query tests are predicate-blind on the SQL shape; doesn't catch `::int` overflow in pathological cases (sums past 2^31). Requires a real DB fixture or generated-SQL assertion to address. Not actionable in this chunk; revisit when a fixture harness is introduced (likely paired with the parked schedule-actions vitest fixture from 0023 follow-ups).

### Codex "Not An Issue" (false-positive reduction)

- CSV mitigation: route handler test exercises the real `buildCsv` → `csvCell` chain end-to-end with a `=SUM(A1:A10)` payload. A future refactor that bypasses csvCell would fail this test.
- Route handler short-circuits on gate failure: confirmed by the new rejection-path test.
