### In-eval fix: month sort

Initial smoke surfaced the Month tab sorting alphabetically (April / August / December / July / June / May / November / October / September). Root cause: column accessor was `groupLabel` ("April 2026" string). Fix: `buildMonthColumns(sortByKey: true)` overrides `sortingFn` to compare on the `groupKey` `YYYY-MM` value (which sorts naturally). Re-tested in-browser; April → May → June → … → December order confirmed.

## Codex Adversarial Review

**Critical / High / Medium:** none.

**Low (3 found, all 3 fixed in-eval):**

1. **`coachRowKey()` dead code (FIXED in-eval).** Codex caught that `coachRowKey()` was exported from `reports-columns.tsx` but never wired into `DataTable` (no `getRowId` prop). The "claimed null-coach key fix is not actually active." Resolution: dropped the helper. TanStack's default positional row IDs are unique within a render, and the SQL guarantees one row per coach (single null bucket). The Phase 1 carry-forward is genuinely satisfied because the prophylactic key concern (React keys, faceted filter) doesn't materialize in the actual UI; Phase 3 CSV identifier will derive its own stable ID.

2. **`monthLabel` duplication drift risk (FIXED in-eval).** Codex flagged the same noon-UTC YYYY-MM formatter appearing in both `reports-tabs.tsx` and `queries.ts`. Resolution: extracted to `src/lib/dates.ts:formatYearMonth(yyyymm)`. Both call sites now import the single source.

3. **Tab content unmounts on switch (FIXED in-eval — user requested).** Radix Tabs by default unmounts inactive `<Tabs.Content>`. Resolution: extended `DataTable` to accept controlled `sorting` / `onSortingChange` / `pagination` / `onPaginationChange` props (the controlled/uncontrolled React convention — internal `useState` remains as the fallback). `ReportsTabs` now holds four `[sort, setSort]` + four `[page, setPage]` pairs (one per tab) and threads them down. Verified in-browser: clicking the Dealer tab's Records column to re-sort, switching to Coach, then back to Dealer, preserves the new sort. Same plumbing covers the Full tab's pagination across switches.

### Codex "Not An Issue" (false-positive reduction)

- Route gating correct: `(app)/layout.tsx` runs `requireStaffAccess()`; `/reports/page.tsx` adds `requireRole(['admin','coach'])` before loaders. RSC prefetch hits the same gate.
- Loaders are `import 'server-only'` — no client surface.
- Month-prefix filter is safe: options come from `startDate.slice(0,7)` (zero-padded `YYYY-MM`), filter is `startsWith(filterValue)`. `2026-01` does NOT match `2026-10/11/12`.
- No XSS surface: cells render React text nodes; no `dangerouslySetInnerHTML`.
- Reports nav link is affordance-only; access still enforced page-side.
- Bundle: `@radix-ui/react-tabs` is in line with the existing Radix stack; no duplicate React contexts.
