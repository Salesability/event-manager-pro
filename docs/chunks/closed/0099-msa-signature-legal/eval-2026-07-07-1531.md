## Codex Adversarial Review

**High:** None.

**Medium:**
1. **Address lines can run off the page in the half-width Client column.** `splitClientAddress` char-caps (120) but doesn't wrap to the 241pt column; a long legal address drawn at `rightColX` could overflow past `rightEdge`. → **FIXED IN-CYCLE (`fa40e74`)**: the renderer now wraps each address line to the column width (`wrap(sanitize(line), 44)`) and bounds the block to 4 visual lines. Verified with a long-address render (wrapped cleanly, anchors stay on-page).
2. **"Full name" not guaranteed** — `contacts.last_name` is `notNull` but imports allow single-token names (`lastName: ''`), so `sendMsaEnvelope` falls back to first-name-only for such a primary contact, preserving the legal failure mode for the adopted-signature default. → **PARKED (0099-a)**: not mechanically fixable (can't invent a surname), and **materially mitigated by design** — the signer-filled `ClientPrintedName` TextBox is the real legal safeguard, and BoldSign lets the signer adopt a full-name signature regardless of the pre-fill. Pre-existing data condition; the chunk still improves on the prior always-first-name behavior.

**Low:**
3. **`CURRENT.md` drift** — still said "Scaffolded — phases not yet started" while the plan is all-Done. → Resolved by the auto-close close-out (rewrites the Active block).

**Verified Not Issues (Codex):**
- `nameBoxY`/`titleBoxY` top-left conversion is consistent with the signature conversion (`pageHeight - (pdfBottomY + fieldHeight)`).
- Under `MAX_ADDRESS_LINES = 4`, the raised `margin + 250` guard leaves room for signature + name + title + email + 4 address lines + authority + footer (no vertical clipping).
- Left/right columns use independent cursors after `underlineY`; the footer is fixed-position and reads no stale shared cursor.
- `FormField.FieldTypeEnum.TextBox` exists in the installed BoldSign SDK; `buildFormField` is field-type-generic.
- MSA send paths pass both new anchors after narrowing to `ok`; there are no other production `sendSignatureRequest` callers, so non-MSA behavior stays signature-only.
