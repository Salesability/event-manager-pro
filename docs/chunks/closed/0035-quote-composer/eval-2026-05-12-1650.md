### Pass 1 fixes (self-resolved, no codex:rescue dispatch)

Both findings were small enough to fix directly in the orchestrator:

1. **High:** Added `isDirty` memo comparing live `inputs`/`taxOverride` against `initial.inputs`/`initial.tax` via flat-shape JSON equality. Send button now disables on dirty with title "Save changes before sending — the email emits the saved quote, not the unsaved edits." A small below-row hint also surfaces: "You have unsaved changes — save before sending." Confirm dialog now reads `persistedLineCount`/`persistedTotal` (sourced from `initial`) instead of the live computed values.

2. **Medium:** Restructured the JSX so the buttons row lives **outside** `<fieldset disabled={isReadOnly}>`. The fieldset now only wraps the inputs panel + line-items table + tax-override input (the actually-disable-needing surface). Save / Send still hide themselves via `!isReadOnly` / `canSend`. Preview's enabled state is determined only by `previewPending`. Verified live: `/quotes/1` (now `sent`) shows the Preview button and the click opens the dialog with the iframe.

## Operator notes (out-of-scope, captured for follow-up)

A user-driven send during this session failed at the GCS upload step:

- `GCS_CREDENTIALS_JSON` in `.env.local` was set to a **file path** (`/...key.json`).
- `src/lib/storage/gcs.ts:39-46`'s `client()` only handles inline JSON: it always calls `JSON.parse(inline)`, which throws on a path and returns `{ error: 'GCS_CREDENTIALS_JSON is not valid JSON.' }`.
- `sendQuote`'s atomic UPDATE flips `status → 'sent'` and sets `pdfStorageKey` **before** the upload call, so the row reached the "stranded sent" state: `status='sent'`, `pdf_storage_key='quotes/1/1.pdf'`, **PDF not in GCS**, **no `quote.sent` audit row**, **no Resend record**. Subsequent Send clicks hit the idempotent short-circuit at `actions.ts:551-554` and returned `{ ok: true }` without retrying the upload.
- The user is renaming the env var to `GOOGLE_APPLICATION_CREDENTIALS=/path/...` (ADC pickup, zero code change — the fallthrough is already wired at `gcs.ts:37-49`). Dev server restart required to re-instantiate the Storage client (module-level cache at `gcs.ts:22-25`).
- Stranded quote #1 needs a manual reset to draft before it can be re-sent: `UPDATE quotes SET status='draft', sent_at=NULL, pdf_storage_key=NULL, updated_at=now() WHERE id=1;`.
- This is the exact failure mode the parked **0026 follow-up (a) degraded-send retry path** was scoped to repair (loud error already in place — "admin repair required" — but no remediation UI yet).

None of the above is a Phase 4 regression; the env-config trap predates Phase 4 and would have affected any prior `sendQuote` call. Surfaced here only because Phase 4 introduced the first UI surface that fires `sendQuote` in normal use.
