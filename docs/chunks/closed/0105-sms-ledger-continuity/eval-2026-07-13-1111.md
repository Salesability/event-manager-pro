## Codex Adversarial Review (verbatim, condensed to findings)

**Medium — key rotation surfaced as "differs", not "unknown"** (`queries.ts`): `loadRecipientHistory` treated any two non-null unequal HMACs as `differs`; after a `SMS_IDENTITY_HMAC_KEY` rotation the same person would get a misleading "name differs from prior sends" warning, contradicting decision.md D2.
**Medium — launch can send a stale recipient list during concurrent re-import** (`actions.ts`): launch evaluated recipients before its transaction while import replaced the list in a separate transaction without sharing the launch advisory lock; snapshots could preserve facts from a superseded list.
**Low — HMAC field separator not collision-safe for names containing `|`** (`identity.ts`): `first|last|phone` concatenation lets `("ann|marie","smith")` collide with `("ann","marie|smith")`.

**Examined, not an issue:** per-call key read (rotation takes effect next revision); nameless rows → null; no readable identity in the permanent ledger or the client payload; dealer scoping of the history join; purge continuity (SET NULL + snapshots, integration-tested); migration 0051 additive-nullable.

## Fixes landed (`4395c1c`)

1. **Fingerprints are key-versioned** — format `<8-hex key id>:<64-hex hmac>`; `compareFingerprints()` reads cross-key comparisons as `unknown` (never a false "differs"), absent values as `unknown`, unversioned values compare directly. Unit-tested incl. rotation.
2. **Import and launch share the campaign advisory lock, and launch evaluates inside its locked transaction** — the list the review promised is the list the send row + snapshots are built from. Residual (a re-import after commit but before dispatch finishes) is the 0103-a durable-dispatch surface, already parked.
3. **Pipes in names fold to spaces in normalization** — the field boundary can't be forged; unit-tested.
